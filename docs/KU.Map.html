Namespace: Map

KU. Map

Contains all map related functions for loading and controlling the map page.

Members

<static> categoriesList :Array.<Object>

List of categories (order that they will be downloaded in) containing the corresponding foursquare ID's.
Type:
  • Array.<Object>

<static> categoryData :Array.<Object>

Actual poi data for each category as determined by the categoriesList. This must be handled dynamically in KU.Map.initializeMap in order to properly determine the serializeID.
Type:
  • Array.<Object>

<static> center :Array.<double>

Center of the map during KU.Map.initializeMap
Type:
  • Array.<double>

<static, constant> debug :boolean

Enables debug messages via alert messages. Do NOT enable this unless there are legitimate problems with plugins (not available to debug via browser).
Type:
  • boolean

<static> downloadIndex :int

The index related to categories for which we are currently downloading or importing from a cached session.
Type:
  • int

<static> footer :boolean

Is the footer showing? The footer contains the POI details and starts off hidden. It is only shown when the user clicks on the POI or searches.
Type:
  • boolean

<static> KEY_WAIT :int

Wait time for incremental searching. This is essentially how long we wait after the user has searched something, before we assume they are done searching.
Type:
  • int

<static> lastValue :string

Represents the last/current value the user has searched for from the free text field / search bar located on the map page.
Type:
  • string

<static> loading :boolean

Tells whether or not map is currently attempting to download or parse article lists. Essentially used to tell whether map is considered to be busy.
Type:
  • boolean

<static> map :Object

The leaflet map object
Type:
  • Object

<static, constant> MAP_FILE_TEMPLATE :int

File template for local storage of foursquare map data for each category. E.g for saving food it would be MAP_DATA_Food filename.
Type:
  • int

<static> markers :Array.<Marker>

Contains all markers/POI's that are added to the map through the latest search results (excluding the current user's location which is not dependent on searching).
Type:
  • Array.<Marker>

<static, constant> MIN_UPDATE_MS :int

This is the minimum time that needs to have passed in order to allow for another foursquare update to both the map and local filesystem.
Type:
  • int

<static> neLat :double

Northeast latitude for map's bounding box. Essentially most northeast latitude for POI's we are concerned with to show the user.
Type:
  • double

<static> neLng :double

Northeast longitude for map's bounding box. Essentially most northeast longitude for POI's we are concerned with to show the user.
Type:
  • double

<static> selectedMarker :Object

Contains the selected marker. This is useful to store so we can easily disable the markers selection.
Type:
  • Object

<static> sentTimeout :Timeout

Contains the last timeout call sent! This allows us to restart the timeout if the user re-searches in any way. The major benefit of this is that it gives us the feeling of incremental searching, e.g we send a timeout of some milliseconds whenever the KEY_UP event triggers, as well as cancelling out the last timeout we sent.
Type:
  • Timeout

<static> serializeID :int

The 'version' of how we save map data to local filesystem. Whenever the structure of this changes, we must change the serialized ID so that we can programmatically know that the old serialized data is not compatible with this new version, and thereby force redownload the new data.
Type:
  • int

<static> swLat :double

Southwest latitude for map's bounding box. Essentially most southwest latitude for POI's we are concerned with to show the user.
Type:
  • double

<static> swLng :double

Southwest longitude for map's bounding box. Essentially most southwest longitude for POI's we are concerned with to show the user.

  • double

Methods

<static> addPOIs(category)

Add POI's of a certain category to the map if it is a match with the user's input through the search bar.
Parameters:
Name Type Description
category Object contains category name and poi array

<static> clearMarkers()

Removes all markers from the map and then clears them from the array.

<static> deselectMarker()

Deselects the currently selected marker from the map. Basically it only switches the icon back to the 'non-selected' icon.

<static> filterMap()

Filters the map POI's to only add markers that the user searched for.

<static> initializeMap()

Initialize the leaflet map with category data and list

<static> readPOIs()

Reads POI's from local file system, starting with the current downloadIndex. Note that if this fails, we will attempt to download the failed category.

<static> showMapDetails(poi)

Shows the footer with POI detail information at the bottom of the map.
Parameters:
Name Type Description
poi Object the informtaion needed to show (name, address, phone etc)

<static> updatePOIs()

Updates POI's by first checking the caches map data files, and if needed re-downloads the POI data from Foursquare and then re-serializes the data to local file system.

Name Type Description

Triggered when the map page is first created based on jQM page create event.
Parameters:
Name Type Description
event Event jQM event, not actually being used by us at the moment.

pageInit

Triggered when the map page is first initialized based on jQM page init event.
Parameters:
Name Type Description
event Event jQM event, not actually being used by us at the moment.

readPOI_Fail

Triggered after a failed read of POI file.

readPOI_Success

Triggered after a successful read of POI file.

searchDirectChangeEvent

Triggered when the user does a direct change. The direct change includes typing then changing focus or pressing the clear button. This is redundant to the incremental search event, except for the clear button!

searchIncrementalChangeEvent

Triggered when the user does a key up event in order to simulate incremental searching for the attached search bar.
