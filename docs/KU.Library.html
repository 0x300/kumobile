Namespace: Library

KU. Library

Contains all library related functions for loading and controlling the library page.

Members

<static> lastAction :string

Represents the last/current action from catalog planet's site. This is essentially a key needed to continue cycling through pages from the most recent search on catalog planets Kettering site. See KU.Library.getNextPage
Type:
  • string

<static> lastValue :string

Represents the last/current value the user has searched for from the free text field / search bar located on the library page.
Type:
  • string

<static> listQueue :Array.<Object>

Contains the current list of article DOM li tag items that still need to be added to the DOM (much faster to add all at once after load is done downloading). This helps prevent the app from seeming to hang or become unresponsive.
Type:
  • Array.<Object>

<static, constant> LOAD_THRESHOLD_PX :int

Designates the minimum number of pixels that the user can scroll (calculated from the bottom) before another load event is triggered.
Type:
  • int

<static> loading :boolean

Tells whether or not library is currently attempting to download or parse article lists. Essentially used to tell whether library is considered to be busy.
Type:
  • boolean

<static> page :int

Stores the last page number downloaded. Note: for library we use the external site at http://catalog.palnet.info instead of Kettering's own site, therefore this page number is only used as a convenience for us. Their system works by a complex hitlist, see KU.Library.getNextPage for more details.
Type:
  • int

<static> reachedEnd :boolean

Tells whether or not the user has reached the end of the scrolling. This is useful mainly to prevent the load to continuously trigger when there are no more results that need to be shown. Obviously this starts off false and is detected during KU.Library.getNextPage.
Type:
  • boolean

<static> sentAjax :Ajax

Contains the last ajax call sent! This allows us to abort the call if the user re-searches in any way (dropdown, or searchbar)
Type:
  • Ajax

<static> sentTimeout :Timeout

Contains the last timeout call sent! This allows us to restart the timeout if the user re-searches in any way (dropdown, or searchbar). The major benefit of this is that it gives us the feeling of incremental searching, e.g we send a timeout of some milliseconds whenever the KEY_UP event triggers, as well as cancelling out the last timeout we sent.
Type:
  • Timeout

<static> type :string

Type of searching for Kettering's library. This basically represents the method of searching that will be used by the system. Note: the naming convention is a bit funky and NOT controlled by KU-Mobile.
Type:
  • string

<static> typing :boolean

Is the user currently typing? This is mainly used to ensure we do not do certain loads or other things when we consider the user is typing.
Type:
  • boolean

Methods

<static> getNextPage()

Downloads the next page in order to gain another 10 articles as part of the article list for the main library page. Note that there are no arguments, but the function uses namespace members.

<static> reinitialize()

Reinitializes all properties of KU.Library as if to restore a new/default instance of the namespace.

Events

checkIScroll

Triggered when iScroll is moved or ended during a scrolling transition. Note this is only used at the moment for older devices which do not support overflow.
Parameters:
Name Type Description
e Object event, not used
d Object contains iscrollview and alike properties

checkScroll

Triggered when regular scroll event happens in library scroller window This should be used for most, or all, modern devices that support it. Note: this detection is setup in KU.ISCROLL using overthrow-detect library.
Parameters:
Name Type Description
event Object event properties, not used

pageCreate

Triggered when the library page is first created based on jQM page create event.
Parameters:
Name Type Description
event Event jQM event, not actually being used by us at the moment.

pageInit

Triggered when the library page is first initialized based on jQM page init event.
Parameters:
Name Type Description
event Event jQM event, not actually being used by us at the moment.

searchDirectChangeEvent

Triggered when the user does a direct change. The direct change includes typing then changing focus or pressing the clear button. This is redundant to the incremental search event, except for the clear button!

searchIncrementalChangeEvent

Triggered when the user does a key up event in order to simulate incremental searching for the attached search bar.

typeChangeEvent

Triggered when the user does a change to the TID/department drop down box. When this happens, we generally need to redo the search.
