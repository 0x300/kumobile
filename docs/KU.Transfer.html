Namespace: Transfer

KU. Transfer

Contains all transfer related functions for loading and controlling the transfer page.

Members

<static> lastCollege :string

Last value the the user searched for by college ID. These college ID's are specified by Kettering and downloaded dynamically with the function KU.Transfer.downloadColleges
Type:
  • string

<static> lastCourse :string

Last value the the user searched for by course id (e.g MATH-102 or MATH102)
Type:
  • string

<static> listQueue :Array.<Object>

Contains the current list of article DOM li tag items that still need to be added to the DOM (much faster to add all at once after load is done downloading). This helps prevent the app from seeming to hang or become unresponsive.
Type:
  • Array.<Object>

<static> loading :boolean

Tells whether or not transfer is currently attempting to download or parse article lists. Essentially used to tell whether transfer is considered to be busy.
Type:
  • boolean

<static> sentAjax :Ajax

Contains the last ajax call sent! This allows us to abort the call if the user re-searches in any way (dropdown, or searchbar)
Type:
  • Ajax

<static> sentTimeout :Timeout

Contains the last timeout call sent! This allows us to restart the timeout if the user re-searches in any way (dropdown, or searchbar). The major benefit of this is that it gives us the feeling of incremental searching, e.g we send a timeout of some milliseconds whenever the KEY_UP event triggers, as well as cancelling out the last timeout we sent.
Type:
  • Timeout

<static> type :string

Type of search, either can be course or college. By course will result in searching by course ID via the search bar, whereas by college will search by college name through college dropdown.
Type:
  • string

<static> typing :boolean

Is the user currently typing? This is mainly used to ensure we do not do certain loads or other things when we consider the user is typing.
Type:
  • boolean

Methods

<static> downloadColleges()

Downloads the list of colleges and their corresponding ID's from Kettering's website/restful API. Then we add this to our dropdown so the user can select his/her preferred college to search by.

<static> downloadCollegeTrans()

Downloads and displays the transfer information by college-ID. Note that this method does not contain any arguments but still uses namespace variables.

<static> downloadCourseTrans()

Downloads and displays the transfer information by course-ID. Note that this method does not contain any arguments but still uses namespace variables.

<static> reinitialize()

Reinitializes all properties of KU.Transfer as if to restore a new/default instance of the namespace.

Events

changeTypeToCollege

Triggered when user chooses the type 'college'. For this we need to rearrange some of the DOM elements (switch between searchbar to a drop down!).

changeTypeToCourse

Triggered when user chooses the type 'course'. For this we need to rearrange some of the DOM elements (switch between drop down to a searchbar!).

collegeDropdownChangeEvent

Triggered when the user does a direct change to the college drop down. When this occurs we need to do another search and display the results.

pageCreate

Triggered when the transfer page is first created based on jQM page create event.
Parameters:
Name Type Description
event Event jQM event, not actually being used by us at the moment.

pageInit

Triggered when the transfer page is first initialized based on jQM page init event.
Parameters:
Name Type Description
event Event jQM event, not actually being used by us at the moment.

searchDirectChangeEvent

Triggered when the user does a direct change to search bar. Includes typing then changing focus or pressing the clear button. This is redundant to the incremental search event, except for the clear button! Note that this is done for searching by course-ID only!

searchIncrementalChangeEvent

Triggered when the user does a key up event in order to simulate incremental searching for the attached search bar. Note that this is done for searching by course-ID only!
